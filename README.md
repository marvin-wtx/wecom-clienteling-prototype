# WeCom Clienteling Prototype Skill · V4.0

V4.0 builds a complete retail clienteling prototype through confirmed stages rather than generating a branded app directly from one line.

1. Run one scope intake and show the exact second-level page list, depth, Journey, and page count.
2. Create business and page-state contracts using common retail structures, authorized mock values, and traceable visible claims.
3. Build and browser-test the neutral mobile product.
4. Confirm one design direction, build representative screens from the executable UI kit and page recipes, pass the Chrome layout probe, and obtain user acceptance.
5. Extend the accepted brand design and add review-only desktop, role, and Journey controls.
6. Bind browser acceptance to the final build hash.

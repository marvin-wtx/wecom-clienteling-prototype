# V4.0 Stage Gates

Fix the earliest failed stage before moving on.

1. **Scope:** one confirmed intake selects roles, modules, second-level pages, page depth, and a primary Journey.
2. **Blueprint:** selected modules use common structures/mock values or source-backed brand facts; unselected modules and unsupported claims are absent.
3. **Page contract:** every selected page has purpose, depth, action, and state; the primary Journey is complete.
4. **Mobile function:** every contracted page and common field is implemented; all visible controls work; native send is direct and its result uses the runtime receipt.
5. **Brand design:** styling follows public or supplied evidence without changing accepted business structure; real or generated assets replace fake art.
6. **Console:** desktop, role, and Journey controls are outside the phone, review-only, and limited to tested mobile capabilities.
7. **Browser:** visible Chrome checks every selected page, primary Journey, scroll, responsive layout, console, and images against the tested build hash.

Do not use QA prose, self-scoring, extra screenshots, or synonym-based blacklists to pass a failed stage.

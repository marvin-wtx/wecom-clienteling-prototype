# Operating-First Layout

Generate layout from the user’s next decision:

1. Put the most time-sensitive daily job and its next action in the first viewport.
2. Keep customer context, task/appointment state, and execution path connected.
3. Use different page structures only when the job differs (triage, dossier, preparation, execution, outcome review).
4. Apply brand visual direction after the workbench reads clearly.

Do not require multiple speculative directions for a sparse brief. Strong agents may explore alternatives privately, but the delivered token should explain the chosen operating loop in plain language. If an authoritative layout is supplied, follow it unless it conflicts with the protected shell or usability.
